# 课题解读

**课题名称：** 面向大语言模型高效推理框架的系统级优化与实现  
**日期：** 2026 年 3 月 21 日

## 一、课题背景

随着大语言模型在边端侧的应用不断拓展，推理服务的重要性日益凸显，其效率直接关乎用户体验，因此，迫切需要对边端侧大模型的推理效率进行优化。

目前，已有推理框架与架构从不同维度对大模型推理展开优化工作。比如：

- **vLLM** 运用 **PagedAttention** 技术，显著提升了显存利用率；
- **llama.cpp** 借助量化技术、硬件加速及跨平台兼容性，有效降低了大模型的部署门槛。

然而，鉴于边端侧环境普遍存在**端到端延迟敏感、资源受限**等特性，现有系统仍需围绕这些痛点，开展更具针对性的优化举措。

本课题要求基于**开源技术栈**部署大模型，分析探讨推理框架不足并进行优化。

### 需要完成

一个具备**端到端推理优化能力**的大模型推理系统。

## 二、课题内容

### 1. 基础功能实现

#### 1.1 模型加载与执行

- 支持至少一个中等规模 LLM（如 **Qwen2.5-14B-Instruct**）
- 支持多种模型格式（**ckpt / GGUF / safetensors** 等）
- 实现**动态 Batch** 与 **Token 缓存机制**

#### 1.2 显存与内存优化

- 实现 **PagedAttention** 或等效机制（显存分层 / 显存分页）
- 支持**激活压缩 / 流式推理**
- 支持量化（至少 **INT8**，建议支持更低位）

### 2. 加分项目

- 支持 **Prompt Cache / KV Cache** 复用
- 在分布式环境部署 **Qwen3.5-122B** 模型
- 开放式研究：分析你使用的推理框架的性能瓶颈，定位并给出分析过程，同时给出可能的优化方向

## 三、评测标准

### 1. 基础功能

需具备完整推理链路，包括：

- 模型加载成功
- Tokenizer 正常工作
- Prefill + Decode 正常执行
- 支持量化
- 支持流式输出

### 2. 优化效果

重点考察以下方面：

- KV Cache 管理
- 显存 / 内存优化
- 动态 Batch / Cache 复用
- 性能瓶颈分析

#### 具体指标

- **TTFT**（首 Token 时延）
- **平均延迟**
- **吞吐**（tokens/sec）
- **KV Cache 占用比例**

### 3. 代码与报告质量

- 代码可实际运行
- 报告清晰展示技术路线及实验效果
