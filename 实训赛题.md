赛题名称:
面向大语言模型高效推理框架的系统级优化与实现（三人组）
赛题介绍:
随着大语言模型（Large Language Models，LLM）在智能问答、代码生成、智能助手等应用中的快速发展，越来越多的应用场景开始尝试将模型部署到边端设备（如个人电脑、边缘服务器或嵌入式设备）上，以实现低延迟响应、减少网络依赖并提升数据隐私保护能力。然而，与云端环境相比，边端设备通常面临算力有限、显存和内存资源受限以及实时性要求更高等挑战，这使得大模型推理的效率优化成为当前人工智能基础设施（AI Infra）领域的重要研究方向。目前已有一些开源推理框架从不同角度对推理效率进行了优化，例如 vLLM 通过 PagedAttention 技术对 KV Cache 进行分页管理，大幅提升显存利用率并支持高并发推理；llama.cpp 则通过模型量化、硬件加速以及跨平台兼容性，使大模型能够在 CPU 或边端设备上运行并降低部署门槛。但总体来看，在资源受限环境下实现端到端高效推理系统仍然具有较大的优化空间。本课题将围绕这一问题展开，要求学生基于开源技术栈设计并实现一个面向边端环境的大语言模型推理框架，通过系统级优化提升推理效率。参与本课题的同学需要具备一定的编程基础（主要为 Python，了解 C/C++ 或 CUDA 将更有帮助），具备基本的深度学习或机器学习知识，并对 Transformer 或大语言模型的基本原理有一定了解，同时熟悉 Linux 开发环境、Git 版本管理以及常见深度学习框架（如 PyTorch）将有助于更好地完成项目。在项目实践过程中，学生需要理解大模型推理的核心流程，包括 Prefill 与 Decode 阶段、Tokenizer、KV Cache 等关键机制，并在此基础上实现一个能够运行的大模型推理系统原型，同时针对推理过程中的性能瓶颈进行优化，例如设计动态 Batch 调度机制、实现 KV Cache 管理策略、探索显存与内存优化方法以及支持模型量化（如 INT8 或更低精度）等。项目还需要通过实验对系统性能进行评估，例如测量首 Token 延迟（TTFT）、整体推理延迟、系统吞吐率（tokens/sec）以及显存占用情况，并对不同优化策略带来的性能提升进行分析。最终，学生团队需要提交可运行的系统代码、系统架构设计说明、实验结果分析报告以及项目展示材料，并在答辩中介绍系统设计思路与优化方法。通过本课题的实践，学生不仅能够深入理解大模型推理的工作机制，还能够掌握 AI 系统开发与性能优化的基本方法，提升在人工智能基础设施与大模型工程方向的实践能力。需要指出的是，由于大模型推理涉及复杂的系统设计和性能调优过程，项目实施过程中可能会面临诸如理解推理流程复杂性、在资源受限环境下运行模型、进行系统性能调优以及适配不同硬件环境等挑战，但通过团队协作与逐步迭代优化，这些问题都可以逐步解决。本课题适合对人工智能系统、大模型技术或系统性能优化方向感兴趣的同学参与，对于希望深入了解 AI Infra 技术并积累工程实践经验的学生具有较高的学习与实践价值。
