# 14B Length-Aware V6.1 初始化

时间：`2026-03-23 05:00:00 +0800`

## 目标

`v6.1` 不改调度框架，只调默认参数。

目的：

- 验证 `v6` 当前的不足是否主要来自参数过硬
- 不覆盖 `v6` 结果
- 保持和 `v6` 同一套 ratio-aware 控制逻辑，便于直接做参数归因

## 相对 V6 的默认参数改动

- `TARGET_SHORT_SHARE_BONUS: 0.20 -> 0.18`
- `MIN_SHORT_SHARE: 0.50 -> 0.42`
- `MAX_SHORT_SHARE: 0.75 -> 0.72`
- `QUEUE_RATIO_CONTROL_GAIN: 1.00 -> 0.80`
- `QUEUE_RATIO_MARGIN: 0.08 -> 0.12`
- `MAX_RATIO_ADJUSTMENT: 0.20 -> 0.12`

## 调参意图

- 降低 `MIN_SHORT_SHARE`
  - 避免 `v6` 中 `target_short_share` 长期被下界卡死在 `0.5`

- 降低 `TARGET_SHORT_SHARE_BONUS`
  - 让基础 short 偏置更温和

- 降低 `GAIN` 与 `MAX_RATIO_ADJUSTMENT`
  - 降低单次修正强度，避免 ratio correction 过冲

- 提高 `QUEUE_RATIO_MARGIN`
  - 扩大死区，减少对轻微比例波动的敏感度

## 运行隔离

- proxy 默认端口：`8086`
- 默认结果目录：`results/vllm_serve_14b_length_aware_v6_1/`
- 默认 `RUN_TAG=14b_length_aware_v6_1`
